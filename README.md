# ECE 563 Final Project: Electric Load Forecasting

This project forecasts hourly electric load for 20 load zones using historical
load data and hourly temperature data from 11 weather stations. The final
prediction target is every hour from June 1-7, 2008 for all 20 zones, written in
the required wide CSV format as `Load_prediction.csv`.

The repository includes both a simple baseline model and a stronger final model:

- `other_model.py`: trains and evaluates the Ridge Regression baseline.
- `best_model.py`: trains and evaluates the selected HistGradientBoosting model,
  then refits on all known non-target data and writes `Load_prediction.csv`.
- `map_tune.py`: discovers the load-zone to temperature-station mapping and runs
  hyperparameter tuning. This script is allowed to take longer than the final
  grading scripts.
- `utils.py`: shared loading, preprocessing, feature engineering, evaluation,
  and CSV-writing helpers.
- `config.py`: project paths, constants, random seed, target date range, and
  default hyperparameters.
- `mapping.json`: selected zone-to-station mapping generated by `map_tune.py`.
- `mapping_diagnostics.json`: diagnostic details for the mapping procedure.
- `best_params.json`: selected hyperparameters and tuning results.
- `tuning_report.md`: readable summary of the tuning experiment.
- `Load_prediction.csv`: generated final predictions for June 1-7, 2008.
- `requirements_summary.md`: structured summary of the course requirements.

## Data

The supplied course data should be placed at:

- `../Load_history_final.csv`
- `../Temp_history_final.csv`

The submitted code folder should sit one level below the two supplied CSV files.

The load and temperature files use a wide hourly format with identifier/date
columns followed by `h1` through `h24`. The final prediction file uses the same
wide hourly structure and contains 140 prediction rows plus the header: 20 zones
times 7 target days.

## Method Summary

Preprocessing converts the wide hourly CSVs to long format, cleans invalid zero
values, imputes local medians, masks robust local load outliers, and adds
calendar, cyclical time, temperature, historical average, last-year, previous-day,
and rolling same-hour load features.

The temperature mapping is generated by `map_tune.py`. For each load zone, it
selects candidate stations using both absolute Pearson correlation and a
RandomForest permutation-importance procedure, then uses the union of those
station selections. The model frame uses mapped station temperature summaries
with regional temperature features as fallbacks.

The baseline model is Ridge Regression with one-hot encoded categorical features
and standardized numeric features. The final model is
`HistGradientBoostingRegressor`, selected by validation RMSE.

Target rows for June 1-7, 2008 are treated as prediction-only and are excluded
from training, validation, testing, preprocessing statistics, and tuning.

## Reproduction

The local environment used during review reported:

```bash
Python 3.10.19
```

Create and activate a Python environment:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Install the required Python packages:

```bash
python3 -m pip install numpy pandas scikit-learn
```

To regenerate the mapping, tuning artifacts, and tuning report from scratch:

```bash
python3 map_tune.py
```

This rewrites `mapping.json`, `mapping_diagnostics.json`, `best_params.json`, and
`tuning_report.md`. It may take several minutes because it evaluates mapping
diagnostics and hyperparameter grids.

To run the baseline model:

```bash
python3 other_model.py
```

To run the selected best model and regenerate the final prediction CSV:

```bash
python3 best_model.py
```

This rewrites `Load_prediction.csv`.

## Splits and Reproducibility

- Random state: `563`
- Tuning split in `map_tune.py`:
  - Training: known non-target load rows from 2004-2006
  - Validation: known non-target load rows from 2007
  - Held-out test: known non-target load rows from 2008, not used for tuning
- Final evaluation split in `other_model.py` and `best_model.py`:
  - Training: known non-target load rows from 2004-2007
  - Held-out testing: known non-target load rows from 2008
- Final prediction refit in `best_model.py`:
  - Training: all known non-target load rows from 2004-2008
  - Prediction: June 1-7, 2008 target rows

## Selected Results

The selected Ridge baseline hyperparameter is:

```text
alpha = 0.001
```

The selected HistGradientBoostingRegressor hyperparameters are:

```text
learning_rate = 0.1
max_iter = 300
max_leaf_nodes = 45
l2_regularization = 0.1
min_samples_leaf = 30
```

Validation results from `tuning_report.md`:

| Model | Validation RMSE | Validation MAPE |
|---|---:|---:|
| Ridge Regression | 280766.30 | 142.01% |
| HistGradientBoostingRegressor | 177694.97 | 31.18% |

The gradient boosting model was selected as the final model because it achieved
the lower validation RMSE.

## Notes

Running `map_tune.py` and `best_model.py` intentionally rewrites generated
artifacts. This is expected for full reproduction. The submitted final model
scripts do not run the expensive tuning loop during normal evaluation; they load
the saved mapping and hyperparameters.
